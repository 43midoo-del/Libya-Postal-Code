# 1-8 مخطط جانت الزمني — الملاحظات والمناقشة

> **المشروع:** تطوير موقع ويب لتوليد وإدارة الرمز البريدي الليبي  
> **القسم:** قسم الحاسوب — كلية العلوم التقنية، جامعة درنة  
> **المرجع:** الفصل الأول — الدراسة التمهيدية (بحث التخرج)

---

## تمهيد

يوضّح **الجدول رقم (1)** مخطط جانت الزمني لمراحل المشروع على مدى اثني عشر أسبوعاً. اللون **البرتقالي** يمثّل المدة **المقترحة** لإنجاز كل مرحلة، بينما اللون **الأخضر** يمثّل المدة **الفعلية** التي استغرقتها المرحلة بعد التنفيذ.

| المرحلة | أ1 | أ2 | أ3 | أ4 | أ5 | أ6 | أ7 | أ8 | أ9 | أ10 | أ11 | أ12 |
|---------|:--:|:--:|:--:|:--:|:--:|:--:|:--:|:--:|:--:|:---:|:---:|:---:|
| التحليل وجمع البيانات | ■ | ■ | ■ | ■ | | | | | | | | |
| تصميم قاعدة البيانات والواجهة | | | ■ | ■ | ■ | ■ | ■ | | | | | |
| برمجة النظام | | | | | | ■ | ■ | ■ | ■ | ■ | | |
| اختبار النظام وكتابة التقرير | | | | | | | | | ■ | ■ | ■ | ■ |

**رموز الألوان في نسخة Word:**

- **برتقالي** — المدة المقترحة (المخطط الأولي)
- **أخضر** — المدة الفعلية (ما تم تنفيذه)

## مناقشة المراحل الأربع

### أولاً: مرحلة التحليل وجمع البيانات

كان من المخطط إنجاز هذه المرحلة خلال **أسبوعين** (الأسبوع الأول والثاني)، ويتضمن ذلك دراسة المشكلة، مراجعة الورقة العلمية السابقة للنظام المكتبي، وجمع متطلبات نظام العنونة البريدية.

إلا أن العمل الفعلي امتد إلى **الأسبوعين الثالث والرابع** نتيجة:

- الحاجة إلى مراجعة أعمق للتقسيم الإداري الليبي (1934 و2007)
- استشارات متكررة مع مشرف المشروع حول صيغة الرمز البريدي
- جمع بيانات جغرافية أولية لمدينة درنة كنموذج تطبيقي

وقد أدّى هذا التأخير إلى تأجيل بداية المرحلة التالية.

| البند | المقترح | الفعلي |
|--------|---------|--------|
| المدة | أسبوعان (1–2) | أربعة أسابيع (1–4) |
| السبب الرئيسي للانحراف | — | جمع البيانات والاستشارات الأكاديمية |

---

### ثانياً: مرحلة تصميم قاعدة البيانات والواجهة

كان مقترحاً البدء في هذه المرحلة من **الأسبوع الثالث** وإنهاؤها في **الأسبوع الرابع**، وتشمل:

- رسم مخطط ERD
- تصميم جداول MySQL
- وضع مسودات الواجهات العربية

لكن بسبب تأخر مرحلة التحليل، وما تبعها من تعديلات على هيكل الرمز الخماسي `X N-M-S P` ودمج ملفات GeoJSON، بدأ التصميم فعلياً في **الأسبوع الخامس** واستمر حتى **الأسبوع السابع**.

وتطلّب الأمر:

- إعادة ضبط العلاقات بين جداول التقسيم الإداري
- إعداد مخططات Use Case وSequence
- مراجعة التصميم بعد الاستشارات والاختبارات الأولية

| البند | المقترح | الفعلي |
|--------|---------|--------|
| البداية | الأسبوع 3 | الأسبوع 5 |
| النهاية | الأسبوع 4 | الأسبوع 7 |
| السبب الرئيسي للانحراف | — | تأخر التحليل + تعديلات الصيغة والخرائط |

---

### ثالثاً: مرحلة برمجة النظام

وفق الخطة الأولية، كان من المفترض أن تبدأ البرمجة في **الأسبوع الخامس** وتنتهي في **الأسبوع التاسع**، وتشمل تنفيذ:

- PHP ونمط MVC
- ربط Leaflet بقاعدة البيانات
- توليد الرمز البريدي آلياً

وبسبب امتداد مرحلة التصميم، انطلقت البرمجة الفعلية متأخرة نسبياً، مع إضافة ميزات لم تكن ضمن المخطط الأولي، منها:

- البطاقة البريدية مع QR
- محرر حدود الشعبيات والمناطق
- تحذيرات مكانية عند خروج النقطة عن حدود الشعبية

ومع ذلك، تم إنجاز المكونات الأساسية ضمن الإطار الزمني المتبقي من الفصل الدراسي:

- تسجيل الدخول والأدوار
- إضافة العناوين وتحديد الموقع على الخريطة
- توليد الرمز والبحث
- لوحة التحكم والإحصائيات

| البند | المقترح | الفعلي |
|--------|---------|--------|
| المدة | الأسابيع 5–9 | الأسابيع 6–10 (تقريباً) |
| السبب الرئيسي للانحراف | — | تأخر التصميم + ميزات إضافية |

---

### رابعاً: مرحلة اختبار النظام وكتابة التقرير

خصّصت الخطة **أربعة أسابيع** (من الأسبوع التاسع إلى الثاني عشر) لـ:

- اختبار الوحدات والتكامل
- اختبار واجهة المستخدم
- إعداد تقرير التخرج

وتشمل الاختبارات:

| الاختبار | الهدف |
|----------|--------|
| عدم تكرار الرمز البريدي | التحقق من `UNIQUE(postal_code)` |
| عدم تكرار الإحداثيات | التحقق من `UNIQUE(latitude, longitude)` |
| الصلاحيات | فصل صلاحيات المدير والموظف والمواطن |
| الأمان | CSRF، جلسات، تشفير كلمات المرور |
| CI الآلي | فحص `php -l` عبر GitHub Actions |

ويُستكمل في هذه المرحلة توثيق النتائج ومقارنتها بأهداف المشروع الستة.

---

## الخلاصة

أظهر مخطط جانت أن المرحلتين **الأولى** و**الثانية** تجاوزتا المدة المقترحة، وهو أمر شائع في المشاريع البرمجية التي تعتمد على جمع بيانات جغرافية واستشارات أكاديمية. ومع ذلك، لم يمنع ذلك إنجاز النظام كنموذج أولي عملي على مدينة درنة، مع إمكانية ضبط الجدول في المراحل اللاحقة عند التوسع الوطني للمشروع.
