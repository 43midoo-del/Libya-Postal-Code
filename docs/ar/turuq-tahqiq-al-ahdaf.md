# 1-4 طرق تحقيق الأهداف

> **المشروع:** تطوير موقع ويب لتوليد وإدارة الرمز البريدي الليبي  
> **القسم:** قسم الحاسوب — كلية العلوم التقنية، جامعة درنة  
> **المرجع:** الفصل الأول — الدراسة التمهيدية (بحث التخرج)

---

## المقدمة

يمكن تحقيق جميع أهداف المشروع الستة المذكورة في **جدول (1-1)** باللجوء إلى مزيج من التقنيات البرمجية والمعمارية والتنظيمية. وفيما يلي الطرق الأساسية المعتمدة:

1. بيئة العمل **XAMPP**
2. لغة **PHP** (الخادم)
3. قواعد البيانات **MySQL**
4. لغات الواجهة الأمامية **HTML** و**CSS** و**JavaScript**
5. المكتبات والأدوات المساعدة
6. تقنية **GIS**
7. نمط **MVC** والأمان

مع ربط كل طريقة بالهدف أو الأهداف التي تخدمها داخل نظام **Libya Postal**.

---

## 1. استخدام بيئة العمل XAMPP

| البند | التفاصيل |
|--------|----------|
| **الأداة** | XAMPP (Apache + PHP 8.2 + MySQL 8) على Windows |
| **الهدف المرتبط** | الهدف 1 (منصة ويب مركزية)، الهدف 2 (معمارية موزعة) |
| **طريقة التحقيق** | تشغيل الخادم محلياً عبر `run-server.bat` أو `php -S 127.0.0.1:8080`؛ نشر مركزي واحد يخدم عدة متصفحات في آن واحد بدلاً من تطبيق مكتبي على جهاز واحد |
| **المخرج** | تطبيق ويب يعمل على `index.php?r=...` مع وصول متزامن للمدير والموظف والمواطن |

**لماذا هذه الطريقة؟**  
النظام المكتبي السابق (C#/Windows) كان مقيداً بجهاز واحد وتلف بسهولة. بيئة الويب المركزية توفّر نقطة دخول واحدة، تحديثاً مركزياً، وقاعدة بيانات مشتركة.

---

## 2. استخدام لغة برمجة صفحات الويب PHP

| البند | التفاصيل |
|--------|----------|
| **الأداة** | PHP 8.2+ (OOP، Namespaces، PDO) |
| **الهدف المرتبط** | الأهداف 1، 3، 5، 6 |
| **طريقة التحقيق** | فصل المنطق في **Controllers** (`AuthController`, `AddressController`, …) و**Models** (`Address`, `User`, …) و**Services** (`PostalCodeService`, `SessionAuth`, `Csrf`) |
| **المخرج** | معالجة النماذج، الجلسات، توليد الرمز، التحقق من الصلاحيات، وعرض صفحات عربية ديناميكية |

**أمثلة تنفيذية في المشروع:**

- `PostalCodeService::formatCode()` — تكوين الرمز بصيغة `B 2-1-S 9`
- `PostalCodeService::reserveNextProperty()` — حجز الرقم التسلسلي داخل معاملة (Transaction)
- `SessionAuth` + `requireAuth()` — عزل الصلاحيات حسب الدور
- `index.php` — موجّه أمامي (Front Controller) لجميع المسارات

## 3. استخدام قواعد البيانات SQL (MySQL)

| البند | التفاصيل |
|--------|----------|
| **الأداة** | MySQL 8، ترميز `utf8mb4_unicode_ci`، محرك InnoDB |
| **الهدف المرتبط** | الهدف 4 (سلامة البيانات)، الهدف 3 (معايير العنونة) |
| **طريقة التحقيق** | تصميم مخطط علائقي هرمي: `states → regions → cities → areas → addresses` مع جدول عدادات `postal_property_counters` |
| **المخرج** | تخزين مركزي آمن للعناوين والتقسيم الإداري والمستخدمين |

**آليات ضمان سلامة البيانات:**

| القيد | الغرض |
|--------|--------|
| `UNIQUE(postal_code)` | منع تكرار الرمز البريدي |
| `UNIQUE(latitude, longitude)` | منع تكرار الموقع الجغرافي |
| `SELECT … FOR UPDATE` على العداد | توليد تسلسلي ذري للعقار دون تعارض |
| `password_hash()` | تخزين كلمات المرور بشكل آمن |
| **Prepared Statements (PDO)** | منع حقن SQL |

**ملفات المخطط:** `database/schema/01_base.sql` + `database/migrations/*.sql` + `database/seeds/*.sql`

## 4. استخدام HTML و CSS و JavaScript (الواجهة الأمامية)

| البند | التفاصيل |
|--------|----------|
| **الأدوات** | HTML5، CSS3، JavaScript (Vanilla — بدون إطار عمل ثقيل) |
| **الهدف المرتبط** | الهدف 5 (تجربة المستخدم)، الأهداف 1، 3، 6 |
| **طريقة التحقيق** | بناء واجهة عربية تفاعلية (RTL) في المتصفح؛ PHP يولّد الهيكل (Views) وJS يتولى التفاعل والخرائط |
| **المخرج** | صفحات سهلة الاستخدام: تسجيل دخول، خريطة، نماذج عناوين، بحث، بطاقة بريدية |

### HTML (هيكل الصفحات)

| الاستخدام | الموقع في المشروع |
|-----------|-------------------|
| قوالب الصفحات العربية | `views/` (مثل `login.php`, `addresses/create.php`, `dashboard/index.php`) |
| أجزاء مشتركة (رأس، تنقّل، تذييل) | `views/partials/` (`head.php`, `main_nav.php`, `flash.php`) |
| نماذج الإدخال | حقول البريد، كلمة المرور، الإحداثيات، التقسيم الإداري |
| إتاحة الوصول | تسميات `label`، `aria`، تخطٍ للمحتوى |

### CSS (التنسيق والمظهر)

| الاستخدام | الموقع في المشروع |
|-----------|-------------------|
| التنسيق العام والواجهة العربية RTL | `css/app.css` |
| صفحة الاستعلام البريدي والبطاقة | `css/postal_lookup.css` |
| محرر الحدود | `css/boundary_editor.css` |
| تقارير العناوين | `css/addresses_report.css` |
| تجاوب الشاشات | Media queries، شبكات بطاقات، جداول منظمة |

### JavaScript (التفاعل والخرائط)

| الاستخدام | الموقع في المشروع |
|-----------|-------------------|
| محرك الخريطة الأساسي | `js/map/core.js` |
| طبقات الشعبيات والمدن | `js/map/shabiyat.js`, `js/map/parcel.js` |
| نماذج إضافة/تعديل العنوان | `js/addresses/form.js`, `js/addresses/save.js` |
| محرر الحدود | `js/boundary/editor.js` |
| مزامنة البلاطات دون اتصال | `js/tile_sync.js` |
| طلبات للخادم | `fetch` لحفظ العناوين وجلب البيانات |

**لماذا هذه الطريقة؟**  
الهدف 5 يتطلب واجهة **عربية سريعة الاستجابة** يفهمها المواطن والموظف. **HTML** يحدد البنية، **CSS** يحقق RTL والتنسيق الموحّد، و**JavaScript** يجعل الخريطة والنماذج تفاعلية دون إعادة تحميل الصفحة في كل خطوة.

**تكامل الواجهة مع الخادم:**

```
المستخدم ← HTML/CSS (عرض) ← JavaScript (تفاعل + خريطة)
                ↕ fetch / POST
         PHP Controllers ← MySQL
```

## 5. استخدام المكتبات والأدوات المساعدة

| المكتبة / الأداة | الاستخدام | الهدف المرتبط |
|------------------|-----------|----------------|
| **Leaflet.js** | خرائط تفاعلية، تحديد الإحداثيات بالنقر | 3، 5 |
| **Leaflet-Geoman** | محرر حدود الشعبيات والمناطق | 3، 6 |
| **GeoJSON** | حدود ليبيا والشعبيات والمدن (`data/`) | 3، 6 |
| **Chart.js** | إحصائيات لوحة التحكم | 5 |
| **QRCode.js** | بطاقة بريدية قابلة للمسح | 5، 6 |
| **html2pdf.js** | تصدير البطاقة البريدية PDF | 5 |
| **Git + GitHub Actions** | التحكم بالإصدارات وفحص `php -l` تلقائياً | 4، 6 |

**طريقة التحقيق:** تحميل المكتبات من مجلد `vendor/` أو CDN؛ ربطها بملفات **JavaScript** الخاصة بالمشروع عبر `fetch` وواجهات JSON داخلية عند الحاجة.

## 6. استخدام تقنية نظم المعلومات الجغرافية (GIS)

| البند | التفاصيل |
|--------|----------|
| **الأداة** | Leaflet + GeoJSON + إحداثيات WGS84 (Latitude / Longitude) |
| **الهدف المرتبط** | الأهداف 3، 5، 6 |
| **طريقة التحقيق** | ربط كل عنوان بإحداثيات على الخريطة؛ دمج التقسيم الإداري الليبي (1934 + 2007) في صيغة الرمز الخماسي `X N-M-S P` |
| **المخرج** | عنوان رقمي مكانياً بديلاً عن الأوصاف الشفهية وصناديق البريد (P.O. Box) |

**خطوات الربط الجغرافي:**

1. المستخدم ينقر على الخريطة → التقاط `(lat, lng)`.
2. التحقق من أن النقطة داخل حدود ليبيا (`GeoBounds` / `GeoPoint`).
3. اختيار الولاية → الشعبية → المدينة → القطاع من القوائم المرتبطة بقاعدة البيانات.
4. توليد الرمز تلقائياً وربطه بالإحداثيات في جدول `addresses`.

## 7. استخدام نمط المعمارية MVC والأمان

| المكوّن | المسار | طريقة تحقيق الهدف |
|---------|--------|-------------------|
| **Model** | `models/` | استعلامات PDO ومنطق البيانات |
| **View** | `views/` | واجهات عربية RTL |
| **Controller** | `controllers/` | معالجة الطلبات والتحقق من الأدوار |
| **CSRF** | `includes/Csrf.php` | حماية النماذج |
| **SessionAuth** | `includes/SessionAuth.php` | مصادقة وجلسات آمنة |

| الهدف | كيف تحققه المعمارية |
|--------|---------------------|
| **5 — تجربة المستخدم** | قوالب موحّدة، تنقّل واضح (`main_nav.php`)، رسائل Flash |
| **2 — معمارية موزعة** | Client (متصفح) ↔ Server (Apache/PHP) ↔ Database (MySQL) |
| **4 — سلامة البيانات** | فصل الطبقات + قيود قاعدة البيانات + أدوار المستخدمين |

## الخلاصة

تحقيق أهداف مشروع العنونة البريدية لا يعتمد على أداة واحدة، بل على **تكامل** بين:

1. **بيئة ويب مركزية** (XAMPP) بدلاً من تطبيق مكتبي معطّل.
2. **لغة PHP** لتنفيذ المنطق والخدمات على الخادم.
3. **قاعدة بيانات MySQL** لضمان الدقة وعدم التكرار.
4. **HTML و CSS و JavaScript** لبناء واجهة عربية تفاعلية في المتصفح.
5. **مكتبات مساعدة** (Leaflet، QR، Chart.js، …) مدمجة عبر JavaScript.
6. **تقنية GIS** لربط العنوان بالموقع الفعلي على الخريطة.
7. **نمط MVC والأمان** لتنظيم الكود وحماية البيانات.

هذا التكامل هو الأساس الذي بُني عليه النظام المنفّذ في مجلد `Projict/` ويُوثَّق بالتفصيل في بقية فصول البحث (التحليل، التصميم، التنفيذ والاختبار).
