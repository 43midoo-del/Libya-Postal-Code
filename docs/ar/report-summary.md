# تقرير 1 — تقدم مشروع نظام إدارة العناوين البريدية (ليبيا)

> **تنبيه:** المذكرة التفصيلية `**مذكرة-التنفيذ-تفصيلية.md`** تضم **كل المراحل** المنفذة (تشمل **البحث/الخريطة/التعديل/الحذف** = roadmap المرحلة 8) بشرح مفصّل يسهّل المناقشة الأكاديمية.

**تاريخ التحديث:** 22 أبريل 2026  
**نطاق التقرير (ملخص):** إنجاز المسار **1–6** + **البحث وعرض النتائج على الخريطة** + **تعديل/حذف (موظف/إدارة)**. التفصيل الممل في `**مذكرة-التنفيذ-تفصيلية.md`**.

## 1. الملخص التنفيذي

جرى إنشاء قاعدة بيانات MySQL وفق الـ SRS، ثم **PDO**، ثم **تسجيل دخول**، ثم **لوحة**، ثم **خريطة Leaflet**، ثم **إضافة عنوان** + **بحث (كود/اسم)** + **مخطط ليبيا** لعدة نتائج + **تعديل/حذف** للموظف/الإدارة. **التفصيل الكامل** في `مذكرة-التنفيذ-تفصيلية.md` (انظر **§7**).

---

## 2. الخطوة 1: قاعدة البيانات (`database.sql`)


| البند              | التفاصيل                                                                                                                                                                               |
| ------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **الملف**          | `database.sql` في جذر المشروع                                                                                                                                                          |
| **قاعدة البيانات** | `libya_postal` (ترميز `utf8mb4` / `utf8mb4_unicode_ci`)                                                                                                                                |
| **الجداول**        | `users`, `states`, `regions`, `cities`, `areas`, `addresses`                                                                                                                           |
| **مفاتيح وقواعد**  | مفاتيح أجنبية بين التقسيم الإداري؛ `users.email` فريد؛ `addresses.postal_code` فريد؛ منع تكرار نفس الإحداثيين عبر `UNIQUE(latitude, longitude)`؛ فهارس داعمة للبحث (`owner_name`, إلخ) |
| **الدور (role)**   | `ENUM('admin','employee','citizen')`                                                                                                                                                   |
| **ملاحظة**         | بذور المستخدم تُنفّذ عبر `database_seed.sql` بعد `database.sql` (انظر الخطوة 3)                                                                                                        |


---

## 3. الخطوة 2: الاتصال بقاعدة البيانات (PHP + PDO)


| البند               | التفاصيل                                                                                                                  |
| ------------------- | ------------------------------------------------------------------------------------------------------------------------- |
| **إعداد الاتصال**   | `config/database.php` — إرجاع مصفوفة (host, port, database, username, password, charset) **يجب تعديلها حسب بيئة التشغيل** |
| **طبقة الاتصال**    | `includes/Database.php` — النطاق `App`، نمط Singleton، `getPdo(): PDO`                                                    |
| **إعدادات PDO**     | `ERRMODE_EXCEPTION`، `FETCH_ASSOC` الافتراضي، `EMULATE_PREPARES = false`                                                  |
| **التحميل**         | `includes/bootstrap.php` — تعريف `APP_ROOT` + autoload + بدء جلسة باسم من `config/app.php`                                |
| **النقطة الرئيسية** | `index.php` — **موجّه أمامي** (معامل `?r=`) يرتبط بالمتحكمات (دخول، لوحة، خريطة، …)                                       |


**أمان / تشغيل:** يُنصح بعدم رفع كلمات مرور قواعد بيانات حقيقية للمستودع؛ اضبط `config/database.php` محلياً.

## 4. الخطوة 3: تسجيل الدخول (ملخص)


| البند       | التفاصيل                                                                                         |
| ----------- | ------------------------------------------------------------------------------------------------ |
| **النموذج** | `models/User.php` — `findByEmail` + تحقق كلمة عبر `password_verify`                              |
| **المتحكم** | `AuthController` (عرض/معالجة دخول، خروج)، `DashboardController` (لوحة مبدئية)                    |
| **الجلسة**  | `SessionAuth` يخزن `id`, `name`, `email`, `role` بعد `session_regenerate_id(true)`               |
| **الحماية** | `Csrf` لنموذج الدخول؛ `BaseController::requireAuth` للوحة                                        |
| **بذور**    | `database_seed.sql` — مستخدم `admin@libyapostal.local` (كلمة المرور: `admin123` **للتطوير فقط**) |


(الشرح المطوّل: انظر `مذكرة-التنفيذ-تفصيلية.md` — القسم 3.)
